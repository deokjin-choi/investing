## Thesis 유형 정의

| 유형 | 핵심 질문 | 적정 보유 기간 |
|---|---|---|
| **재평가형** | 시장이 이 기업을 잘못 보고 있고, 그 인식이 언젠가 바뀔 것인가? | 1~3년 |
| **사이클형** | 이 산업의 장기 사이클이 바닥이고 상승 구간이 남아있는가? | 3~5년 |
| **단기촉매형** | 특정 이벤트가 주가를 단기적으로 움직일 것인가? | 3~9개월 |
| **\[옵션\] 사이클 저점 집중** | 자산군(코인 포함)이 전고점 대비 −75% 이하 구간인가? | 상황별 판단 |

Thesis가 살아있는 한 보유, 소진되거나 반증 조건이 충족되면 매도.

## 폴더 구조

```
investing/
├── README.md                        # 이 파일 — 현황 진입점
├── investor-profile.md              # Agent 기본 참조 (투자 성향·재무 상황·원칙)
│
├── positions/                       # 현재 보유 종목
│   └── qualcomm/
│       ├── thesis.md               # 현재 thesis 요약 (항상 최신 상태 유지)
│       ├── log.md                  # 매매 의사결정 로그 (왜 샀고 왜 팔았는가)
│       └── [날짜별 분석 노트]
│
├── candidates/                      # 매수 검토 중인 종목
│   └── naver/
│       └── [날짜별 탐색 노트]
│
└── research/                        # 섹터·매크로 일반 연구 (종목 비특정)
    └── [날짜별 메모]
```

**운영 원칙:**
- `thesis.md`: 해당 종목에 대한 현재 판단을 항상 최신으로 유지. 노트가 쌓여도 이 파일만 보면 현황 파악 가능.
- `log.md`: 매수·추가·비중 조절·매도 등 주요 결정 시점마다 기록. 체결 단가·수량은 MTS 참조, 여기엔 **왜 그 결정을 내렸는가**만 기록.
- 종목이 매도 완료되면 `positions/` → `archive/` 로 이동.
